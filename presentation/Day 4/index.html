Day 4

Performance Optimization

Memoization, Rendering, and React Compiler

Measure, then optimize

Slide 1 – Why Performance Optimization Matters

  • Large component trees and frequent re-renders
  • Expensive calculations or passing functions as props
  • Slow UI interactions
  • React is fast, but unnecessary re-renders hurt

Slide 2 – How React Renders (Quick Refresher)

  • A component re-renders when its state changes
  • When its props change
  • When its parent re-renders
  • By default: parent render triggers child renders

Slide 3 – What Is an Unnecessary Re-render?


function Parent() {
  const [count, setCount] = useState(0);
  return (
    <>
      
      
    
  );
}

HeavyComponent re-renders even though it does not use count. Optimization helps.

Slide 4 – Optimization Tools Overview

ToolPurpose
React.memoPrevent unnecessary re-renders
useMemoCache expensive calculations
useCallbackCache function references

Slide 5 – What Is React.memo?

  • Higher-order component
  • Memoizes a component
  • Skips re-render if props did not change

const MyComponent = React.memo(Component);

Slide 6 – React.memo Example


const Greeting = React.memo(function Greeting({ name }: { name: string }) {
  console.log("Greeting rendered");
  return 
Hello {name}
;
});

Re-renders only if name changes.

Slide 7 – Without vs With React.memo

  • Without: parent renders → child renders always
  • With: parent renders → child renders only if props changed
  • React.memo uses shallow prop comparison

Slide 8 – When React.memo Helps

  • Heavy components or long lists
  • UI components with stable props
  • Not useful when props change often or component is cheap
  • Rule: measure first, optimize second

Slide 9 – What Is useMemo?

  • Memoizes a value
  • Avoids recalculating expensive logic on every render

const value = useMemo(() => compute(), [deps]);

Slide 10 – Problem: Expensive Calculation


function Component({ items }: { items: number[] }) {
  const total = items.reduce((sum, n) => sum + n, 0);
  return 
Total: {total}
;
}

Recalculated on every render even if items did not change.

Slide 11 – useMemo Example


const total = useMemo(() => {
  return items.reduce((sum, n) => sum + n, 0);
}, [items]);

Recalculates only when items changes.

Slide 12 – What useMemo Actually Does

  • Caches the result, not the function
  • Dependency array controls recomputation
  • Wrong dependencies cause bugs

Slide 13 – When to Use useMemo

  • Expensive calculations used in rendering
  • Dependencies are stable
  • Skip for simple calculations or premature optimization

Slide 14 – Why useCallback Exists

  • Functions are recreated on every render
  • New function references make props appear changed

const handleClick = () => setCount(count + 1);

Slide 15 – useCallback Explained


const fn = useCallback(() => {
  /* logic */
}, [deps]);

Memoizes a function to keep the same reference between renders.

Slide 16 – useCallback Example


const handleClick = useCallback(() => {
  setCount(prev => prev + 1);
}, []);

Stable function reference; great for memoized children.

Slide 17 – React.memo + useCallback Together


const Button = React.memo(({ onClick }) => {
  return ;
});

const onClick = useCallback(() => {
  setCount(c => c + 1);
}, []);

Common pattern: memoized child + stable handler.

Slide 18 – useMemo vs useCallback

useMemouseCallback
Memoizes a valueMemoizes a function
Returns the resultReturns the function
useMemo(() => value)useCallback(() => fn)

useCallback(fn) is roughly useMemo(() => fn).

Slide 19 – Common Mistakes

  • Overusing memo hooks
  • Missing dependencies
  • Optimizing too early
  • Memoizing everything "just in case"

Slide 20 – Performance Rules of Thumb

  • React is fast by default
  • Optimize only real problems
  • Use DevTools Profiler

Slide 21 – Real-World Example (Summary)

  • List component → React.memo
  • Expensive filtering → useMemo
  • Event handlers → useCallback
Module 5 — Performance

React Compiler

🔧 Compile-time tool

Automatically optimizes re-renders at build time.

💤 Less boilerplate

Reduces the need for manual useMemo and useCallback.

🔍 Transparent

Works behind the scenes — no API changes needed.

Memoization applied automatically at compile time.

Module 5 — Performance

Before the Compiler


// Parent.tsx
function Parent({ value }: { value: number }) {
  const expensive = useMemo(() => {
    return heavyCalculation(value);
  }, [value]);

  return <Child data={expensive} />;
}

Manual useMemo adds cognitive overhead and boilerplate to every component.

Slide 24 – Example: With React Compiler


function Parent({ value }) {
  const expensive = heavyCalculation(value);
  return ;
}
  • No useMemo needed
  • Same performance with cleaner code
  • Compiler tracks dependencies and memoizes

Slide 25 – How to Add the React Compiler

1. Upgrade & Install

npm install react@latest react-dom@latest
npm install babel-plugin-react-compiler

2. Update Config

Add the babel plugin to your Vite config.

vite.config.ts
import react from "@vitejs/plugin-react";

export default {
  plugins: [
    react({
      babel: {
        plugins: ["babel-plugin-react-compiler"],
      },
    }),
  ],
};

💡 Pro Tip: Once configured, the compiler automatically memoizes components and hooks—reducing the need for useMemo and useCallback.
