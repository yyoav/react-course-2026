Rendering Lists with .map() & Keys (40 min)

Why .map() is used in React

  • In React, you rarely manually create repetitive HTML. Instead, we use JavaScript arrays and .map() to generate lists dynamically.
  • .map() iterates through an array and returns a JSX element for each item.
  • Example analogy: Think of a bakery conveyor belt – each item (JSX) is created from a recipe (array item).

The importance of keys

  • Keys help React identify which items changed, were added, or removed.
  • Without keys, React might re-render the entire list unnecessarily, causing performance issues or even bugs.
  • Keys must be unique among siblings and ideally stable across renders.
  • Example: Using index as a key is okay for static lists but not for dynamic lists where items may move or be deleted.

Best Practices for Keys

  • Prefer unique IDs over array indexes (id from API or DB).
  • Ensure keys do not change between renders unless the item is truly new.
  • Avoid using random numbers because they defeat React's diffing optimization.

💡 Show a visual comparison: list without keys vs list with keys

Example: Rendering a Fruit List

JavaScript/JSX 
const fruits = ["Apple", "Banana", "Cherry"];

function FruitList() {
  return (
    <ul className="fruit-list">
      {fruits.map((fruit, index) => (
        <li key={index}>{fruit}</li>
      ))}
    </ul>
  );
}
CSS 
.fruit-list {
  list-style: none;
  padding: 0;

  li {
    padding: 0.5rem 1rem;
    background: #f0f0f0;
    margin-bottom: 0.5rem;
    border-radius: 6px;
  }
}

Conditional Rendering (30 min)

What is conditional rendering?

  • Conditional rendering is the ability to show or hide components or elements based on certain conditions (state, props, or data).
  • It is similar to if/else statements in programming but adapted for JSX.

Ternary Operator (? :)

  • Syntax: condition ? renderThisIfTrue : renderThisIfFalse
  • Use for choosing between two options.
  • Example: Show "Welcome back" if logged in, else show "Please log in".

Logical AND (&&)

  • Syntax: condition && renderThisIfTrue
  • Use for rendering something only if a condition is true.
  • Example: Show "Premium badge" only for premium users.

Why conditional rendering matters

  • Improves UX by showing context-sensitive content.
  • Avoids unnecessary DOM elements and reduces render load.

💡 Show side-by-side code example of ? : vs && and explain when each is preferable

Example: Conditional Greeting

JavaScript/JSX 
function Greeting({ isLoggedIn }) {
  return (
    <div className="greeting">
      {isLoggedIn ? (
        <h1>Welcome back!</h1>
      ) : (
        <h1>Please log in</h1>
      )}
      {isLoggedIn && (
        <p>You have new notifications!</p>
      )}
    </div>
  );
}
CSS 
.greeting {
  h1 {
    color: #0070f3;
  }

  p {
    color: #555;
    font-style: italic;
  }
}

Reconciliation & Virtual DOM (20 min)

What is the Virtual DOM?

  • React maintains a Virtual DOM, a lightweight copy of the real DOM stored in memory.
  • Think of it as a blueprint of the UI React can compare quickly.

How React updates the UI efficiently

When a component state or prop changes:

  1. React creates a new Virtual DOM tree.
  2. Compares it to the previous Virtual DOM (this is called diffing).
  3. Only updates the actual DOM elements that changed, instead of re-rendering everything.

Analogy: Instead of repainting an entire wall, React only touches the bricks that are cracked.

Importance of keys in reconciliation

  • Keys allow React to match elements between renders, making diffing fast and accurate.
  • Without keys, React may re-render the whole list even if only one item changed.
Diagram Concept: State Change → New Virtual DOM → Diff → Minimal DOM update → Render

Reconciliation in Action

Example Concept

If a list of 10 items changes only the 3rd item,
React updates only that element, not all 10.

Rules of Hooks Overview (10 min)

Why hooks exist

Hooks let us use state and lifecycle features in functional components, which previously were only available in class components.

Rules of Hooks

  1. Call Hooks at the top level: Don't put Hooks inside loops, conditions, or nested functions. This ensures that Hooks are called in the same order every render.
  2. Call Hooks only in React functions: Only inside functional components or custom hooks.
  3. Hooks order must stay the same: Changing order will break React's internal tracking of state and effects.

Common mistakes

  • Calling useState inside if statements.
  • Using Hooks inside non-component functions.
❌ Wrong 
if (loggedIn)
  const [state, setState] = useState();
✅ Right 
const [state, setState] = useState();
if (loggedIn) { ... }

useEffect for Side Effects (50 min)

What is a side effect?

Any operation that interacts outside the component or affects something outside its scope.

Examples: Fetching data, timers, subscriptions, manual DOM manipulation.

When does useEffect run?

  • After the component renders.
  • With dependencies, it runs only when those dependencies change.
  • Empty dependency array [] → runs once after first render (similar to componentDidMount in class components).

Cleanup function

  • Return a function from useEffect to clean up resources like intervals, subscriptions, or event listeners.
  • Prevents memory leaks.

💡 Show example with timer: effect sets interval → cleanup clears interval

Example: Clock with useEffect

JavaScript/JSX 
import { useState, useEffect } from "react";

export default function Clock() {
  const [time, setTime] = useState(new Date());

  useEffect(() => {
    const timer = setInterval(() => {
      setTime(new Date());
    }, 1000);

    return () => clearInterval(timer);
  }, []);

  return (
    <h1>{time.toLocaleTimeString()}</h1>
  );
}
CSS 
h1 {
  font-size: 2rem;
  color: #333;
}

Custom Hooks (30 min)

What is a custom hook?

  • A JavaScript function that starts with use and reuses component logic.
  • Can use other hooks inside it (useState, useEffect, etc.).

Why use custom hooks

  • Avoid repeating code.
  • Separate logic from UI.
  • Makes code cleaner and easier to test.

Example

useFetch encapsulates data fetching logic, so any component can just call const { data, loading } = useFetch(url).

💡 Show example: component without custom hook (repeated fetch code) vs component with custom hook

Example: useFetch Custom Hook

import { useState, useEffect } from "react";

export function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch(url)
      .then((res) => res.json())
      .then((json) => setData(json))
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading };
}

useRef for Mutable References & DOM Access (20 min)

What is useRef?

  • A hook that holds a mutable object whose .current property persists across renders.
  • Can store DOM elements or mutable values.

Use cases

  • DOM access: Focus input, scroll to element, measure element size.
  • Mutable storage: Store previous state or timer ID without triggering re-renders.

Important rules

  • Updating .current does not trigger a render.
  • Useful for performance-sensitive operations.

💡 Show visual: component renders, useRef stores a DOM element → manipulate it without re-rendering

Example: Focus Input with useRef

JavaScript/JSX 
import { useRef } from "react";

function FocusInput() {
  const inputRef = useRef();

  const handleFocus = () => {
    inputRef.current.focus();
  };

  return (
    <div className="focus-container">
      <input
        ref={inputRef}
        type="text"
        placeholder="Type here..."
      />
      <button onClick={handleFocus}>
        Focus Input
      </button>
    </div>
  );
}
CSS 
.focus-container {
  display: flex;
  gap: 0.5rem;

  input {
    padding: 0.5rem;
    border-radius: 4px;
    border: 1px solid #ccc;
  }

  button {
    padding: 0.5rem 1rem;
    background: #0070f3;
    color: white;
    border: none;
    border-radius: 4px;
  }
}
